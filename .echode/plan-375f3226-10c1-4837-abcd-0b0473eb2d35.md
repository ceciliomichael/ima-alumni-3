# UI Text Update and Standardized Required Field Indicators

## Overview
This plan addresses the user's request to update the Landing Page text and standardize the visual indication of mandatory fields (red asterisks) across the application. To adhere to strict software quality standards (DRY, SOLID), we will introduce a reusable `FormLabel` component instead of scattering raw CSS and HTML updates across multiple files.

## Architectural Changes
- **New UI Component**: Create `src/components/ui/FormLabel` to encapsulate label logic and styling. This promotes reusability and consistency.
- **CSS Modularization**: Styles for the label will be co-located with the component, avoiding global CSS pollution in `index.css`.

## Tasks

### 1. Create Reusable FormLabel Component
Create a new directory `src/components/ui/FormLabel` with the following structure:
- **`FormLabel.tsx`**: A strongly-typed React component that accepts `required`, `className`, and standard label attributes. It will automatically append the red asterisk when `required` is true.
- **`FormLabel.css`**: Contains the specific styles for the asterisk (`::after` pseudo-element).
- **`index.ts`**: Barrel export for clean imports.

### 2. Update Landing Page
- **File**: `src/pages/Landing/LandingPage.tsx`
